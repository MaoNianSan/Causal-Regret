# Toy delayed-feedback diagnostic

This synthetic appendix experiment isolates one mechanism: feedback generated by an action at source time may arrive under a different decision state. The `naive` learner deliberately binds arrival-time feedback to the current action; `causal_labelled` uses the source action carried by the feedback item. Performance is evaluated using structural-time causal regret, not arrival-time reward.

## Modes

```bash
python reproduce_fast.py
python reproduce_full.py
# Alias for the full paper-oriented pipeline
python reproduce_paper.py
# Optional source-code consistency check
python code_check.py
```

`fast` uses seeds `[0, 1, 2]`, writes raw diagnostic logs, generates summaries and figures, and then runs the strict self-check. `full` uses the complete configured seed list and retains lightweight summaries, figures, and execution logs. Use `outputs/full/` only for candidate paper artifacts after its check passes.

## Direct commands

```bash
python main.py --mode fast
python self_check.py --mode fast
python main.py --mode full
python self_check.py --mode full
```

The runner is self-contained. It does not require an external `common.dual_mode` package.

## Required success conditions

A valid run has `logs/run_metadata.json` with `status: success` and `backend_status: executed`; nonempty seed, method, and trajectory summaries; figure source CSVs and figures; and, in fast mode, raw delay, arrival, step, and diagnostic logs. `self_check.py` rejects skipped backends, missing/empty artifacts, nonfinite values, invalid shared delay paths, incomplete design coverage, nonzero oracle regret, and zero-delay disagreement between naive and causal-labelled learners.

## Scope and limitations

This is a mechanism diagnostic, not a real-data study, a proxy-sufficiency test, a complete delayed-bandit benchmark, or a replacement for the main experiments. The causal-labelled learner is intentionally a simple EWMA learner; remaining regret relative to the oracle reflects learning and information limits rather than a failure of source attribution alone.

## GitHub packaging notes

### Purpose

Minimal synthetic delayed-feedback recovery demonstration.

### Directory layout

- `src/`: artifact contract, runner, figure, and summarization helpers.
- `ipy/`: GitHub-facing result-check notebook.
- `outputs/`: saved lightweight summaries, figures, and logs.
- `input/`: optional local-only placement holder.

### Input data requirement

No external raw dataset is required.

### How to run fast validation

Use `python reproduce_fast.py` or `python main.py --mode fast`, then `python self_check.py --mode fast`.

### How to run full reproduction

Use `python reproduce_full.py` or `python reproduce_paper.py`, then `python self_check.py --mode full`. This was not executed during final packaging.

### How to inspect existing results

Open `ipy/toy_result_check.ipynb` or inspect `outputs/full/summary/`, `outputs/full/figures/`, `outputs/full/logs/`, and `output_manifest.md`.

### Expected outputs

Expected GitHub-facing outputs are trajectory figures, figure source CSV files, method/seed/trajectory summaries, logs, and manifest files.

### What is committed to GitHub

Source code, README, notebook, manifest, lightweight summaries, figures, and logs.

### What remains local and why

Runtime caches, raw regenerated logs not needed for inspection, and temporary files remain local because they are reproducible or noisy.
